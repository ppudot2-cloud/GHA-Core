🔵 Pipeline 1 — build-and-deploy.yml
⚡ Trigger: workflow_dispatch  ·  inputs: solutions, mock_deploy, checker_error_level, base_solutions, enable_backup
🔍
Setup
job: setup
Resolves the solution build/deploy matrix and evaluates credential presence flags.
▶
Trigger condition
Always runs — first job, no needs
needs: (none)runs-on: ubuntu-latest
Steps
1
Checkout repository
Checks out GHA-Dynamics at current ref (main or the branch that triggered dispatch).
actions/checkout@v4
2
Checkout GHA-Core CI scripts
Checks out GHA-Core repo (using GHA_CORE_PAT) into .ci/ so PS1 scripts are accessible to subsequent steps.
actions/checkout@v4 · path: .ci
3
Resolve solution matrix
Runs Resolve-SolutionMatrix.ps1 with the dispatch input (solutions, PP_SOLUTION_NAME var). Performs topological sort from dependsOn fields. Emits matrix, solution_list, solution_count to GITHUB_OUTPUT.
.ci/.github/scripts/dynamics/Resolve-SolutionMatrix.ps1
4
Check credential presence
Evaluates PP_APP_ID, PP_CLIENT_SECRET, PP_TENANT_ID, GHA_CORE_PAT secret presence in a run: step (secrets.* only accessible here, not in reusable workflow with: blocks). Emits *_set boolean flags.
Outputs
matrix — JSON array of solution objects with name, source_folder, data_schema_file etc.
solution_list — comma-separated solution names
solution_count — integer count
pp_app_id_set, pp_secret_set, pp_tenant_set, gha_core_pat_set — 'true'/'false' strings

⚡ export jobs run in parallel (one per solution, fail-fast: false). commit job runs after ALL export jobs complete.
📤
Export | <SolutionName> (parallel × N)
job: export (matrix) — _stage-export.yml
Exports each solution from the sandbox environment and uploads unpacked source as an artifact.
▶

Steps
1
Reveille
Checks out GHA-Dynamics + GHA-Core (.ci/). Optionally logs into Azure Key Vault to resolve secrets (jfrog_enabled=false here). In mock mode, skips Azure login.

2
Install PAC CLI
Installs Power Platform CLI via microsoft/powerplatform-actions/actions-install. Adds pac to PATH.
pac-install@main

Export Solution
Real: exports unmanaged ZIP from PP_SDBX_URL, unpacks to export-src/{solution_name}/. Mock: logs simulation, reuses existing src/solutions/{name} tree.

if mock_deploy=true → skip real export, reuse committed source

Uploads the unpacked source tree as exported-source-{name} artifact (retention: 1 day). Build stage downloads this to pack into ZIPs.

🔵 Pipeline 1 — Stage Build
⚡ build jobs run in parallel per solution. validate job runs after ALL build jobs complete (needs: build, if: build succeeded).
🏗️
Build — <SolutionName> (parallel × N)
job: build (matrix) — _stage-build.yml → _job-build.yml
Packs ZIPs, runs Solution Checker, exports config data, uploads artifacts to GitHub and JFrog.
▶
Trigger condition
needs: setup, stage-export · always runs after export completes
needs: setup, stage-exportfail-fast: false
Steps (inside _job-build.yml)
1
Reveille
Checkout GHA-Dynamics + GHA-Core. Azure Key Vault login (if jfrog_enabled). Resolves JFROG_TOKEN.
reveille@main
2
Install PAC CLI
Power Platform CLI for solution pack operations.
pac-install@main
3
Download exported source artifact
Downloads exported-source-{name} from earlier export job. Only when use_exported_source=true (combined pipeline). Release pipeline mode skips this — packs directly from committed src/solutions/.
if: inputs.use_exported_source == true
4
Resolve source folder
Sets effective source folder: export-src/{name}/ (export mode) or src/solutions/{name}/ (release mode).
5
Pack Solution
Runs pack-solution action: pac solution pack to produce {name}_unmanaged.zip and {name}_managed.zip. Stamps build version. Outputs: version, artifact_name, unmanaged_zip, managed_zip.
pack-solution@main
6
Solution Checker
Always on — cannot be skipped in real mode. Runs Microsoft Solution Checker against the managed ZIP. Geo hardcoded to UnitedStates. Minimum severity configured by checker_error_level input. Uploads SARIF report. Fails build if violations exceed threshold.
solution-checker@main · geo: UnitedStates (hardcoded)
7
Export Config Data
Only runs if solution has a dataSchemaFile configured in solutions.json. Exports configuration migration data (records, not schema) from source environment. Real mode: powerplatform-actions/export-data. Mock mode: Export-ConfigDataSim.ps1 creates placeholder ZIP. Skip if no schema defined.
export-config-data@main
if: data_schema_file != ''
8
Upload solution artifact for deploy
Uploads the packed ZIPs + config-data ZIP + deployment-settings JSON as solution-artifact-{name} (retention 7 days). This is the handoff artifact for all deploy jobs.
actions/upload-artifact@v4 · name: solution-artifact-{name} · retention: 7d
9
JFrog Upload
Uploads unmanaged ZIP, managed ZIP, and SARIF report to JFrog Artifactory under {repo}/{github-repo}/{solution}/{run}-{attempt}/. Provides indefinite retention and prodDeployed tagging. Calls Invoke-JFrogAction.ps1 -Action upload. Mock: skipped.
jfrog-upload@main · Invoke-JFrogAction.ps1
10
Write build summary
Calls Write-BuildSummary.ps1 to append a per-solution build table to GITHUB_STEP_SUMMARY and write a job-summary-{name}.json to a job-summaries/ artifact for the consolidated pipeline summary.
Write-BuildSummary.ps1
Artifact outputs
solution-artifact-{name} — ZIPs + config-data + deployment-settings (7d retention)
job-summary-{name} — JSON record for pipeline summary
